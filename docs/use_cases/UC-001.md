# Use Case: Sign In

## Overview

**Use Case ID:** UC-001  
**Use Case Name:** Sign In  
**Primary Actor:** Member  
**Secondary Actor:** Librarian  
**Goal:** Authenticate with the system to access role-appropriate features  
**Status:** Approved

## Preconditions

- The actor has a user account created by the librarian.
- The actor is not currently signed in.

## Main Success Scenario

1. Actor navigates to the application.
2. System displays the login form with username and password fields.
3. Actor enters their username and password.
4. Actor submits the form.
5. System validates the credentials.
6. System redirects the actor to the home page appropriate for their role.

## Alternative Flows

### A1: Invalid Credentials

**Trigger:** Username does not exist or password does not match (step 5).  
**Flow:**

1. System displays an error message indicating the credentials are incorrect.
2. System clears the password field and keeps focus on the form.
3. Actor may try again; use case continues at step 3.

### A2: Empty Fields Submitted

**Trigger:** Actor submits the form with one or both fields empty (step 4).  
**Flow:**

1. System displays a validation message indicating required fields are missing.
2. Use case continues at step 3.

## Postconditions

### Success Postconditions

- Actor is authenticated and has an active session.
- Actor is on the home page for their role (Member sees catalog; Librarian sees active loans).

### Failure Postconditions

- No session is created.
- System displays the login form with an appropriate error message.

## Business Rules

### BR-001: Account Creation by Librarian Only

User accounts cannot be self-registered. Only a librarian can create accounts, so the system provides no sign-up link on the login page.

### BR-002: Role-Based Redirect

After successful sign-in, Members are redirected to the catalog view. Librarians are redirected to the active loans view.

### BR-003: Generic Error Message

The error message on invalid credentials must not reveal whether the username exists or the password was wrong, to prevent username enumeration.
